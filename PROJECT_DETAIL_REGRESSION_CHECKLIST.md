# 项目详情页专项回归清单

目标：每个 Tab 都覆盖“加载→编辑→保存→只刷新本切片不丢上下文”，避免项目详情页成为最大回归风险点。

## 通用检查

- 打开项目详情后，当前项目、当前 Tab、滚动位置和右侧操作菜单不应被无关刷新打断。
- 新增、编辑、删除后，只刷新对应数据切片，不强制回首页。
- 保存失败时保留用户输入，并展示明确错误。
- 无数据时显示可行动空状态，而不是空白。

## Tab 清单

- 甘特图：加载项目甘特、全局甘特入口可见、无任务时显示空状态。
- 项目脉搏：燃尽图、快照、偏差入口可见，缺历史数据时提示如何生成。
- 阶段/任务：加载阶段，新增任务，切换完成状态，保存后只刷新阶段区。
- 里程碑：新增、完成、回顾，庆祝提示不影响当前 Tab。
- 团队/联系人：新增、编辑、删除，权限字段不丢。
- 接口规范：加载内置标准、上传对方文档、快速对比、AI 助手可打开。
- 接口流/设备：新增设备、更新状态、关联床位，保存后局部刷新。
- 问题：新增问题、编辑状态、企微推送、查看推送回执。
- 离场：申请离场、记录返场，项目状态同步但不丢上下文。
- 工作日志：手动日志、AI 填报、一句话上报、批量导入。
- 文档/费用：上传/下载/删除文档，费用新增/编辑/统计。
- 变更/验收/满意度：新增、审批、状态更新，保存后停留原 Tab。
- 沟通/会议助手：保存沟通记录，会议助手生成任务/问题后刷新任务与问题。
- 依赖/关键路径：新增依赖、查看关键路径、影响分析可打开。
- 站会/日报：生成纪要、推送企微、历史可查。
